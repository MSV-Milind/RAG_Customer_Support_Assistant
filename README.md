# RAG Customer Support Assistant

A Retrieval-Augmented Generation (RAG) system that answers support queries
from a PDF knowledge base, orchestrated with a LangGraph 2-node workflow and
HITL (Human-in-the-Loop) escalation.

## Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Embeddings | `all-MiniLM-L6-v2` (local) | No API cost, fast, 384-dim vectors |
| Vector DB | ChromaDB | Lightweight, file-based, zero infra |
| LLM | Groq — `llama3.3-70b-Versatile` | Sub-second latency, free tier |
| Orchestration | LangGraph | Explicit state machine, easy HITL |

## Project Structure

```
rag_support_project/
├── data/               ← Put your PDF(s) here
├── chroma_db/          ← Auto-generated by ingestion (git-ignored)
├── docs/               ← HLD, LLD, Technical Documentation
├── logs/
│   └── escalations.log ← HITL escalation records
├── src/
│   ├── config.py       ← All tuneable parameters
│   ├── ingest.py       ← Phase 1: PDF → ChromaDB
│   ├── retriever.py    ← Phase 2: similarity search
│   ├── graph.py        ← Phase 3: LangGraph nodes & routing
│   └── utils.py        ← Prompts, parsers, logger helpers
├── main.py             ← Entry point
├── .env.example        ← Copy → .env, fill in GROQ_API_KEY
└── requirements.txt
```

## Quick Start

```bash
# 1. Clone / unzip the project
cd rag_support_project

# 2. Create virtual environment
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Set your Groq API key
cp .env.example .env
# Edit .env and paste your GROQ_API_KEY

# 5. Place PDF(s) in data/
#    e.g. data/wikipedia_ai.pdf

# 6. Run ingestion (one-time, or when you add new PDFs)
python main.py --ingest

# 7. Start the interactive assistant
python main.py

# Or one-shot:
python main.py --query "What is reinforcement learning?"
```

## Configuration

All parameters are in `src/config.py`:

| Parameter | Default | Description |
|---|---|---|
| `CHUNK_SIZE` | 700 | Characters per chunk |
| `CHUNK_OVERLAP` | 75 | Overlap between adjacent chunks |
| `TOP_K` | 4 | Max chunks returned per query |
| `SIMILARITY_THRESHOLD` | 0.30 | Min cosine similarity score to include a chunk |
| `GROQ_MODEL` | `llama3-8b-8192` | Swap for `llama3-70b-8192` for more accuracy |

## Swapping the Knowledge Base

1. Delete or clear `data/` and `chroma_db/`.
2. Place new PDFs in `data/`.
3. Run `python main.py --ingest`.

## HITL Escalation

When the LLM cannot answer (or the query is sensitive), it outputs `ESCALATE`.
The graph router catches this, logs the full state to `logs/escalations.log`,
and returns a human-readable escalation message to the user.
